The [stow tree](stow-tree.md) links plain files from the live repo — but a
file that must embed a `/nix/store` path (a plugin path, a wrapped binary, a
rendered option value) would go stale the moment the store path changes.
Those files are instead **generated by the owning feature module**
(`pkgs.writeText` or rendered from module options) and symlinked into place,
so every rebuild refreshes them atomically. The link mechanism is per-class —
darwin: `system.activationScripts.postActivation.text = lib.mkOrder 1600`
(after the stow restow at 1500), run as the user with `ln -sfn`; nixos:
`systemd.tmpfiles.rules` — a `d` rule pre-owns the directory as the user and
an `L+` rule replaces any stale link declaratively. When the feature is a
[cross-OS twin](cross-os-module-twins.md), non-trivial generated text is
shared via a `lib/` builder
([`lib/direnv-nom-wrapper.nix`](../../lib/direnv-nom-wrapper.nix)).

The split in practice:

- [tmux](../modules/tmux.md): static config is stowed; `plugins.conf` — which
  references plugin store paths — is generated and linked next to it by each
  twin ([`modules/darwin/tmux.nix`](../../modules/darwin/tmux.nix) ↔
  [`modules/nixos/tmux.nix`](../../modules/nixos/tmux.nix)).
- [ghostty](../modules/ghostty.md): the stowed config ends with
  `config-file = ?os.conf`; each twin generates its own OS half of the file
  (window chrome and font size on macOS; Hyprland/GTK integration on Linux).
- [direnv-nom](../modules/direnv-nom.md): `zz-nom-wrapper.sh` bakes nom and
  closure-diff store paths, generated from the shared `lib/` builder and
  linked per class.
- [claude-account-selector](../modules/claude-account-selector.md): the zsh
  wrapper is a plain file, but its host-specific rule preamble is rendered
  from Nix options and prepended via `writeText`.
- [podman-desktop](../modules/podman-desktop.md): GUI settings that reference
  packaged binaries are generated; the rest is stowed.

Rule of thumb: **if the file's content depends on evaluation output, the
module owns it; otherwise the stow tree owns it.** Each generated file lives
beside its stowed siblings so the app sees one coherent config directory.
