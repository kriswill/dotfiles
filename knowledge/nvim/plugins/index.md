# plugins

Catalog of every Neovim plugin — one doc per spec file under
`lua/plugins/`. Each spec
returns `{ src, version?, trigger, deps?, setup }` and is dispatched by the
[plugin architecture](../architecture.md); load order is pinned in
`lua/config/plugins.lua`. Stubs are generated by `okf scaffold` — enrich by
hand, and when a plugin is removed delete its doc here too.

## Concepts

* [AdvancedNewFile.nvim](advanced-new-file.md) - Create new files with interactive path completion, auto-creating intermediate directories.
* [baleia.nvim](baleia.md) - Renders ANSI escape codes as real colors — automatic for *.log buffers, on demand via :BaleiaColorize.
* [blink.cmp](blink-cmp.md) - Completion engine (super-tab preset) over LSP/path/snippets/buffer with lazydev ranked first; cmdline completion with ghost text.
* [colorful-winsep.nvim](colorful-winsep.md) - Colors the separator around the active split in kanagawa oniViolet; also publishes the palette as _G.kanagawa_colors.
* [kanagawa.nvim](colorscheme.md) - The kanagawa "wave" colorscheme, loaded first and transparent, with markdown highlight-link overrides.
* [nvim-dap (debug)](debug.md) - DAP debugging with dap-ui and the Go adapter — loads only on first debug keypress; UI auto-opens on launch and closes on exit.
* [direnv.nvim](direnv.md) - direnv integration — manual allow/deny/reload under <leader>fe, statusline indicator, no autoload.
* [efmls-configs-nvim](efmls-configs-nvim.md) - Prebuilt linter/formatter tool definitions for efm-langserver — a data dependency of lsp/efm.lua, no setup of its own.
* [fidget.nvim](fidget.md) - LSP progress spinner in the corner — stock setup, no options.
* [gitsigns.nvim](gitsigns.md) - Git hunk signs in the sign column — stock setup, no options.
* [nvim-highlight-colors](highlight-colors.md) - Inline virtual-text color previews for hex/tailwind values, plus custom palettes — EFG design-system tokens and the full tmux 256-color set.
* [lazydev.nvim](lazydev-nvim.md) - Neovim-API aware lua-language-server workspace for config development; ft-lazy on lua, luv types on vim.uv.
* [lualine.nvim](lualine.md) - Global statusline themed by kanagawa, built from lualine-so-fancy components (mode, diagnostics, branch/diff, LSP servers).
* [mini.splitjoin](mini-splitjoin.md) - Split/join argument lists across lines with sj/sk; the default toggle mapping is disabled in favor of explicit directions.
* [oil.nvim](oil-nvim.md) - Directories as editable buffers, opened floating with `-`; hidden files shown, icons via mini.icons.
* [schemastore.nvim](schemastore.md) - JSON/YAML schema catalog consumed by the yaml language server (no setup — a pure data library).
* [snacks.nvim](snacks.md) - The utility platform — picker, explorer, dashboard, terminal, zen, notifier — owning most of the leader keymap surface, plus a custom vim.pack plugin-inventory picker.
* [nvim-tmux-navigation](tmux.md) - One set of keys (<C-h/j/k/l>) to move across nvim splits and tmux panes, including from terminal mode; disabled when a pane is zoomed.
* [nvim-treesitter-context](treesitter-context.md) - Sticky scope header (up to 3 lines, cursor mode) pinned to the top of the window; toggled with <leader>ut.
* [nvim-treesitter-textobjects](treesitter-textobjects.md) - Treesitter text objects (af/if, ac/ic, ao, as) and parameter swapping, main branch, keymaps set explicitly.
* [nvim-treesitter](treesitter.md) - Treesitter parsers on the main branch (0.12+ API) — async install, highlight+indent per filetype, TSUpdate wired to PackChanged.
* [vim-sleuth](vim-sleuth.md) - Auto-detects per-buffer indentation from file content and neighbors — overrides the 2-space defaults where projects differ.
* [which-key.nvim](which-key.md) - Keymap popup (helix preset, 300 ms) that names the leader namespaces; <leader>? shows buffer-local maps.
